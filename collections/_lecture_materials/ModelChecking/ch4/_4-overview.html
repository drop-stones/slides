Overview

In Ch.3, we models systems as Kripke structures.

In Ch.4, we discuss temporal logic that specify the dynamic behavior of systems.

Temporal Logic can specify the following properties:
  • next : a property holds in next time, after the one computation.
    • ex) If \( a \) holds, \( c \) will hold in next time.
  • eventually : a property holds in the future.
    • ex) The program will eventually finish.
  • never : a property never holds.
    • ex) An error state is never reached.
In this chapter, we focus on the following logics in temporal logics:
  • CTL* (Computation Tree Logic)
  • CTL
  • ACTL* (Universal CTL)
  • LTL (Leanear Temporal Logic)
